## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Test Login For Automation

You can enable a dedicated test login page for browser or AI walkthroughs:

```bash
ENABLE_TEST_LOGIN=true
TEST_LOGIN_EMAIL=your-test-user@example.com
TEST_LOGIN_PASSWORD=your-test-password
```

With those env vars set, visit [http://localhost:3000/test-login](http://localhost:3000/test-login).
The page signs in with Supabase using the configured test account and then redirects into the app.

Keep this disabled in production unless you explicitly want it exposed there.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.
